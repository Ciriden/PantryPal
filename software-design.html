Software Design Document

This document outlines the design approach and architecture for the project.

  • Pantry Pal SRS File

System Architecture

The system follows a client-server architecture with the frontend (user interface) hosted as a web application, and the backend (server-side logic) managed via a RESTful API.

Design Patterns

  • Model-View-Controller (MVC) for clean separation of concerns in the codebase
  • Observer Pattern for handling real-time notifications
  • Singleton Pattern for managing database connections

Database Design

The database will consist of the following main tables:

  • Users Table (Stores user details and roles)
  • Tasks Table (Stores task details, due dates, and status)
  • Notifications Table (Stores real-time notification logs)
